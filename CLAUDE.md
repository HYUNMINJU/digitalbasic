# 디지털 기초 수업 실습 앱 (digitalbasic)

발달장애 성인 대상 **인공지능 수업**의 실습 웹앱.
본론은 3차시(AI가 배우는 방법)이고, 1·2차시는 학습자가 컴퓨터 조작 자체를 어려워해서
앞에 깔아 둔 준비운동이다 — 이 순서를 뒤집지 말 것.

1차시: 전원 켜고 끄기 · 마우스 · 키보드. 2차시: 키보드 위치 찾기 · 따라 쓰기.
3차시: 지도학습 / 비지도학습 체험. 4차시: AI와 대화 (대화창 사용법 + 낱말로 답 고르는 원리).

## 핵심 파일

- `index.html` — 차시 선택 화면 (1~4차시 카드). 리다이렉트 아님.
- `lesson1.html` — 1차시 (단일 자체완결 HTML, 인라인 CSS/JS/SVG).
- `lesson2.html` — 2차시. lesson1의 셸(헤더·TTS·볼륨·진행 점·칭찬·축하)을 복사해 만들었고, 스텝 타입은 `info`/`findkey`(위치 찾기)/`typeword`(따라 쓰기) 세 개.
  - `typeword`: 목표는 완성형(`도리`)으로 보여주고 입력칸은 실시간 조합(`composeHangul`). 틀린 자모도 **입력을 허용**해 빨갛게 남기고, 지우기로 바로잡아야 완성된다(`S.typed` 배열이 상태).
  - **단어 추가 시 금지**: 복합모음(ㅘㅙㅚㅝㅞㅟㅢ)과 된소리(ㄲㄸㅃㅆㅉ)·겹받침. 화면 키보드에 없고 Shift도 안 쓰므로 못 친다. `사과`(ㅘ)·`코끼리`(ㄲ)는 조합이 깨진다. 단자음+단모음만 쓸 것 — `고양이`·`대한민국`은 정상. 파트(모듈) 2개: '키보드 위치 찾기'·'글자 따라 쓰기'. 내부 홈(카드 선택) 화면에 index.html로 나가는 '다른 수업 고르기' 링크.
- `lesson3.html` — 3차시. 원안은 `웹 실습 페이지 구현 계획.docx`. 스텝 타입 5개:
  `info`/`pick`(지도학습)/`sort`(비지도학습)/`compare`/`feature`/`recap`. 파트 3개.
  - 사진은 **두 세트**다. `PHOTOS_SUP`(`images/ai01~08.jpg`, 강아지·고양이)는 지도학습용,
    `PHOTOS_UNSUP`(`images/un01~08.jpg`, 토끼·햄스터)는 비지도학습용.
    **같은 사진을 재사용하지 말 것** — 지루하기도 하지만, 더 나쁜 건 학습자가 앞 단계 답
    ("강아지 여기, 고양이 저기")을 그대로 되풀이해서 '비슷한 점 찾기'를 안 하게 된다는 점이다.
  - 두 세트 모두 **종류와 털색(밝음/어두움)을 일부러 교차**시켰다.
    한쪽으로만 깔끔하게 나뉘게 바꾸면 비지도학습이 '정답 맞추기'로 변질되고 `feature` 질문이 무의미해진다.
  - 비지도 세트는 정답이 새지 않아야 한다: `alt`는 "동물 사진 N", 이름표 빈 칸,
    파일명도 중립(`un01`, `rabbit01` 아님). 스크린리더로도 종류를 알 수 없어야 한다.
  - 사진이 없으면 `onerror`로 이모지 카드 대체 → 이미지 없이도 실습이 끝까지 돌아간다.
  - `sort`: **끌어다 놓기 + 누르기 둘 다** 지원. 드래그는 발달장애 학습자에게 어려우니
    누르기(그룹 버튼 메뉴)를 절대 없애지 말 것. 12px 미만 움직임은 클릭으로 처리(손떨림).
  - 원안 중 **일부러 안 따른 것**: 파일 3분할(→단일 HTML), '올바르게 선택한 수' 표시(→점수 금지),
    숫자 단축키·Esc(→인지 부담). 원안의 "지체장애인" 대상 서술도 이 수업 실제 대상과 다르다.
- `lesson4.html` — 4차시. 스텝 타입 3개: `info`/`send`(대화)/`recap`. 파트 3개.
  - **API를 쓰지 않는다 (되돌리지 말 것).** 학생들이 각자 노트북으로 접속하고 이 앱은
    GitHub Pages 정적 사이트다. API 키를 넣으면 F12로 노출되고 소스도 공개라
    누구나 키를 가져가 **선생님 계정에 요금이 청구된다.** 조심해서 될 문제가 아니다.
    게다가 진짜 AI는 속을 볼 수 없어서 「원리 가르치기」라는 목표 자체를 못 이룬다.
  - 답변은 `RULES` 배열의 **단순 부분 문자열 검색**. 일부러 단순하게 뒀다 — 복잡하면 원리가 안 보인다.
  - `send`의 `mode`가 두 개: `cards`(문장 카드 고르기)·`fill`(빈칸에 낱말 직접 치기).
    **'쓰기 → 보내기'가 두 동작임을 익히는 게 1파트의 핵심** — 카드를 눌러도 전송되지 않는다.
  - `reveal:true`면 보낸 말에서 걸린 낱말을 노랗게 표시하고 무엇을 봤는지 설명한다(2파트 핵심).
  - **`<input>`을 쓰지 않는다.** 2차시처럼 화면 자모 키보드 + `composeHangul`로 조립한다.
    진짜 input을 쓰면 한/영이 영문일 때 `날씨` 대신 `skfTl`가 찍히고 학생은 이유도 모른 채 막힌다.
  - **`fill`의 `word`·카드 문장을 바꿀 때 반드시 확인할 것**:
    ① `word`는 화면 키보드에 있는 자모만 — 된소리(ㅆ)·복합모음(ㅘ) 금지.
       `날씨`로 뒀다가 ㅆ에서 막히는 걸 실제로 겪었다(`노래`로 교체).
    ② `unknown` 파트 카드는 `RULES`의 어떤 낱말도 들어 있으면 안 된다.
       `우리 집 강아지 이름은?`이 `이름` 규칙에 걸려 AI가 아는 척 답했다.
- `마우스배우기.html` — 폐기된 옛 버전. 수정하지 말 것 (참고용).

각 차시는 독립된 자체완결 파일. 셸 코드가 중복되지만 의도된 선택 — 한 차시 수정이 다른 차시를 깨뜨리지 않는다.
(3차시만 `images/`를 쓴다. 사진은 400px·JPEG로 줄여 16장 합계 232KB — 원본 1024px PNG는 장당 1.3MB라 수업 노트북에 무겁다.)

## 코딩 규칙

- ponytail 플러그인 스타일: 외부 라이브러리 0, 브라우저 기본 기능만 (Web Speech API TTS, localStorage, CSS 애니메이션). 최소 코드.
- 구조: `MODULES` 배열(단일 데이터 소스) + `RUN` 스텝 타입 디스패처. 화면은 `#stage` 하나를 innerHTML로 재렌더.
- 새 연습 추가 = MODULES에 스텝 객체 추가 + (새 타입이면) RUN에 러너 함수 추가.

## 접근성 원칙 (변경 금지)

- 실패 상태 없음: "틀렸어요" 금지, 오입력은 부드러운 안내 음성만
- 한 화면 한 행동, 큰 타깃(88px+), 큰 글씨, 고대비 밝은 테마(빔프로젝터용)
- 모든 지시문 TTS 자동 낭독(15자 내외) + 헤더 소리 조절(🔊/🔉/🔇)·다시 듣기
- 시간 압박 없음, 타깃 위치 랜덤 배제, 성공마다 칭찬
- 키보드 판정은 `e.key`가 아닌 **`e.code`** 기준 (한글 IME 대응)

## 로컬 테스트

- Playwright MCP는 file:// 차단 → `python -m http.server 8737` 후 `http://localhost:8737/lesson1.html`
- TTS는 헤드리스에서 안 들림 → 콘솔 `[tts]` 로그로 문구 확인
- 확인 크기: 1920×1080(프로젝터) / 1366×768(수업 노트북) / 1024×600
  - **1024×600이 관문**: 2차시 화면 키보드는 5행(숫자행 포함)이라 세로가 빠듯하다. 키보드 하단이 잘리지 않는지 반드시 확인.
  - **1366×768도 같이 볼 것**: 3차시 CSS breakpoint가 `max-height:860px`인 이유다.
    760으로 두면 768 화면이 스타일을 못 받아 그냥 넘친다(실제로 겪은 버그).
  - 3차시는 `sort` 화면이 제일 빡빡하다(사진 8 + 그룹칸 2 + 버튼). 넘침 확인은 이렇게:
    `document.querySelector('main#stage')`의 `scrollHeight - clientHeight`가 0이어야 한다.
  - 4차시는 **대화 기록이 길어져도 입력칸·보내기·키보드가 화면에 남아야** 한다.
    `.chatlog`만 `overflow-y:auto`로 스크롤된다. 말풍선을 10개쯤 넣고 확인할 것.
- 파일을 고쳐도 브라우저가 옛 파일을 물고 있는 일이 잦다 → **`?v=숫자`를 붙여** 확인.
- 진행 기록: localStorage `digibasic_l1_v1`~`digibasic_l4_v1`(차시별 모듈 완료), `digibasic_vol`(소리 설정, 차시 공유)
