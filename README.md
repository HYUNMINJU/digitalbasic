# 디지털 기초 수업 실습 앱

발달장애 성인 대상 **인공지능 수업**에서 학생들이 각자 노트북으로 직접 실습하는 웹앱입니다.

수업의 본론은 **3차시(AI가 배우는 방법)**입니다. 학습자들이 컴퓨터 조작 자체를 어려워해서
1·2차시(전원·마우스·키보드)를 준비운동으로 앞에 두었습니다.

- **학생 접속 주소**: https://hyunminju.github.io/digitalbasic/
- 차시별 단일 HTML 파일, 외부 라이브러리 없음, 인터넷만 되면 어떤 브라우저에서든 동작
- 접속하면 차시 선택 화면(`index.html`)이 나오고, 거기서 1~3차시를 고른다

## 1차시 — 컴퓨터와 친해지기 (`lesson1.html`)

| 모듈 | 단계 |
|---|---|
| ⏻ 전원 켜고 끄기 (7단계) | 전원 버튼 찾아 누르기 → 부팅 기다리기 → 바탕화면 알기 → "끌 때는 화면에서!" 개념 → 시작 버튼 → 전원 → 시스템 종료 |
| 🖱️ 마우스 연습 (8단계) | 역할 소개 → 별에 올리고 3초 기다리기 → 클릭 → 더블클릭(선물 열기) → 오른쪽 클릭(비밀 메뉴) → 스크롤(꿀벌 옮기기) → 드래그(과일 4개) → 자유 연습장 |
| ⌨️ 키보드 연습 (5단계) | 역할 소개 → 아무 키나 눌러보기 → 스페이스 → 엔터 → 내 이름 첫 글자 자유 놀이 |

세 모듈을 모두 완료하면 "오늘 공부 다 했어요! 🎓" 축하 화면이 나옵니다.

## 2차시 — 키보드와 친해지기 (`lesson2.html`)

화면 키보드에서 목표 키가 파랗게 반짝이면, 실제 키보드에서 같은 자리를 찾아 누른다.
"화면 ↔ 실물 대응" 한 가지 기술에 집중한다. 두 파트로 나뉜다.

| 파트 | 단계 |
|---|---|
| 🔍 키보드 위치 찾기 (13단계) | 소개 → 스페이스·엔터 → 가운데줄(ㅏㅇㄴ) → 윗줄(ㄱㅂ) → 아랫줄(ㅡㅊ) → 숫자(1·5·0) |
| ✍️ 글자 따라 쓰기 (10단어) | 소개 → 가·누·말·벗·아가·도리 → 나무·사람 → 고양이 → 대한민국 |

**따라 쓰기 동작**: 목표는 완성된 글자(`도리`)로 보여주고, 아래 작은 자모열(`ㄷ ㅗ ㄹ ㅣ`)이 누르는 순서를 안내한다.
입력칸은 치는 대로 실시간 조합된다(`ㄷ` → `도` → `도리`). 잘못 누른 글자는 **빨갛게 남고**, 이때 키보드에서
**지우기 ⌫ 키가 반짝여** 지우도록 안내한다. 목표와 정확히 일치해야 다음으로 넘어간다.

> 단어 추가 시 주의: 화면 키보드에 복합모음(ㅘㅙㅚ…)과 된소리(ㄲㄸㅃ…)가 없고 Shift도 쓰지 않는다.
> `사과`·`코끼리` 같은 낱말은 조합이 깨지므로 **단자음+단모음** 낱말만 쓸 것.

## 3차시 — AI는 어떻게 배울까요? (`lesson3.html`)

"정답을 알려주는 학습"과 "정답 없이 비슷한 것끼리 묶는 학습"의 차이를 직접 겪어 본다.
세 파트로 나뉜다.

| 파트 | 단계 | 사진 |
|---|---|---|
| 🏷️ 정답을 알려주며 가르치기 (지도학습) | 소개 → 강아지만 고르기 → 고양이만 고르기 → 정리 | 🐶🐱 강아지·고양이 8장 |
| 🧩 비슷한 것끼리 모으기 (비지도학습) | 소개 → 8장을 두 무리로 나누기 → 정리 | 🐰🐹 토끼·햄스터 8장 |
| ⚖️ 두 가지를 비교해요 | 소개 → 지도/비지도 비교표 → 무엇을 보고 나눴나(복수 선택) → 정리 | — |

**두 파트에 서로 다른 동물을 쓴다.** 같은 사진을 다시 쓰면 지루하기도 하지만, 더 큰 문제는
학습자가 앞에서 낸 답("강아지 여기, 고양이 저기")을 그대로 되풀이해서 정작 *비슷한 점 찾기*를
안 하게 된다는 점이다.

**사진은 일부러 애매하게 만들었다.** 두 세트 모두 종류와 털색(밝음/어두움)이 교차하도록
생성해서, 학습자가 *종류로도* *색깔로도* *자세로도* 나눌 수 있다. 그래야 비지도학습이
"정답 맞추기"가 되지 않고, 마지막 "어떤 특징을 보았나요?" 질문과 **"사람마다 중요하게 본
특징이 다를 수 있어요"**라는 마무리가 살아난다.

> 비지도학습 화면에서는 정답이 새지 않는다 — 이름표도, 대체 텍스트("동물 사진 3")도,
> 파일 이름(`un01.jpg`)도 종류를 드러내지 않는다.

**나누는 방법 두 가지** — 사진을 아래 무리로 **끌어다 놓거나**, 그냥 **눌러서** 나오는
버튼으로 고른다. 드래그는 손 움직임이 불편하면 어려우므로 누르기를 반드시 함께 둔다.
12px 미만으로 움직이면 손떨림으로 보고 클릭으로 처리한다.

> 사진(`images/`)을 못 불러와도 이모지 카드로 자동 대체되어 실습이 끊기지 않는다.

## 설계 원칙

- 실패 없음: "틀렸어요" 표시나 점수·감점 없이 부드러운 음성 안내로만 교정
  - 2차시 따라 쓰기의 빨간 글자는 "실패"가 아니라 **어디를 고치면 되는지 알려주는 표시**다.
    지우고 다시 쓰면 그만이고, 몇 번을 틀려도 기록에 남지 않는다.
- 한 화면 한 행동, 큰 타깃(88px+)·큰 글씨·고대비(빔프로젝터 대응)
- 모든 지시문 음성(TTS) 자동 낭독 + 소리 크게/작게/끄기 + 다시 듣기
- 시간 압박 없음, 성공마다 칭찬, 모듈 완료 배지 저장
- 정답 위치 랜덤 배제 — 매번 같은 순서라 교사가 미리 안내할 수 있고 학생도 예측할 수 있다
- 한글 IME 상태와 무관하게 키보드 판정(`e.code` 기준), 터치패드 완전 지원
